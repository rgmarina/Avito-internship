# Тест-кейсы для микросервиса объявлений

## Создание объявления (POST /api/1/item)

### Позитивные тест-кейсы:
**TC01:** Создание объявления с валидными данными - **FAILED**
- Входные данные: корректные sellerID, name, price, statistics.
- Ожидаемый результат: статус 200, возврат созданного объявления с ID.
- Фактический результат: в ответе отсутствует поле id.

**TC02:** Создание объявления только с обязательными полями.
- Входные данные: только обязательные поля: sellerID, name, price.
- Ожидаемый результат: статус 200.
- Фактический результат: PASSED.

### Негативные тест-кейсы:
**TC03:** Создание объявления без обязательных полей.
- Входные данные: отсутствует sellerID.
- Ожидаемый результат: статус 400.
- Фактический резульата: PASSED.

**TC04:** Создание объявления с невалидным типом данных.
- Входные данные: sellerID как строка, likes как строка.
- Ожидаемый результат: статус 400.
- Фактический результат: PASSED.

**TC05:** Создание объявления с отрицательной ценой.
- Входные данные: price = -100.
- Ожидаемый результат: статус 400.
- Фактический результат: PASSED.

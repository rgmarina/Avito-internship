# Баг-репорты микросервиса объявлений Авито

## Баг №API-01

**Заголовок:** API не возвращает ID созданного объявления в ответе на POST /api/1/item.

**Описание:** При успешном создании объявления (HTTP 200) API не возвращает поле `id` в ответе, что блокирует выполнение любых последующих операций с созданным объявлением.

**Серьезность:** Критическая

**Приоритет:** Высокий

**Шаги воспроизведения:**
1. Выполнить POST запрос на /api/1/item с валидными данными.
2. Получить ответ со статусом 200.
3. Проверить наличие поля `id` в теле ответа.

**Ожидаемый результат:**
```json
{
    "id": "9bcf0ced-8478-4dc2-a483-98a5f885054c",
    "sellerID": 123456,
    "name": "Test Item",
    "price": 1000,
    "createdAt": "2024-01-01T00:00:00Z"
}
```

**Фактический результат:**
```json
{
    "status": "Сохранили объявление - 9bcf0ced-8478-4dc2-a483-98a5f885054c"
}
```

**Затронутые тест-кейсы:**
- TC001: Создание объявления с валидными данными.
- TC006: Получение существующего объявления.
- TC009: Получение объявлений существующего продавца.
- TC012: Получение статистики существующего объявления.
- TC014: Полный цикл создание->получение->проверка.
- TC015: Создание нескольких объявлений одного продавца.
- TC016: Проверка граничных значений sellerID.

## Баг №API-02

**Заголовок:** API возвращает статус 400 вместо 404 для несуществующих ресурсов.

**Описание:** При обращении к несуществующим объявлениям через GET /api/1/item/{id} и GET /api/1/statistic/{id} API возвращает статус 400 (Bad Request) вместо корректного 404 (Not Found).

**Серьезность:** Critical

**Приоритет:** High

**Шаги воспроизведения:**
1. Выполнить GET запрос на /api/1/item/nonexistent_id_12345.
2. Проверить код статуса ответа.
3. Выполнить GET запрос на /api/1/statistic/nonexistent_stat_id_123.
4. Проверить код статуса ответа.

**Ожидаемый результат:** HTTP статус 404 Not Found для обоих запросов.

**Фактический результат:** HTTP статус 400 Bad Request для обоих запросов.

**Затронутые тест-кейсы:**
- TC007: Получение несуществующего объявления.
- TC013: Получение статистики несуществующего объявления.
---
## Баг №API-03

**Заголовок:** Некорректная валидация граничных значений sellerID.

**Описание:** API принимает значение sellerID = 111110, которое находится ниже минимального допустимого значения (111111), и возвращает статус 200 вместо ошибки валидации.

**Серьезность:** Серьезная

**Приоритет:** Средний

**Шаги воспроизведения:**
1. Выполнить POST запрос на /api/1/item с sellerID = 111110.
2. Проверить код статуса ответа.

**Ожидаемый результат:** HTTP статус 400 Bad Request с сообщением о невалидном sellerID.

**Фактический результат:** HTTP статус 200 OK (ошибочное принятие невалидного значения).

**Затронутые тест-кейсы:**
- TC016: Проверка граничных значений для sellerID.
